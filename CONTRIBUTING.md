# Contributing to ThreadSync Simulator

Thank you for your interest in contributing! We welcome improvements to the simulator's logic, UI, and educational content.

## How to Contribute

1. **Fork the Repository**: Create your own copy of the project.
2. **Clone Locally**: `git clone https://github.com/Khyati-tech22/ThreadSync-Simulator.git`
3. **Create a Branch**: `git checkout -b feature/your-feature-name`
4. **Make Changes**: Ensure your code follows the existing style (see `.prettierrc`).
5. **Test**: Run the app locally and verify the simulation behavior.
6. **Commit**: Use descriptive commit messages (e.g., `feat: add priority scheduling`).
7. **Push & Pull Request**: Push to your fork and submit a PR to the `main` branch.

## Areas for Improvement

- **Scheduling Algorithms**: Implement Round-Robin (RR) or Shortest Job First (SJF) visualizations.
- **IPC Demos**: Add Message Passing or Pipe simulation modules.
- **Performance**: Optimize the SharedArrayBuffer sync loop.

Happy coding!
